1. What is Software Testing and why is it important?

Answer:

Software Testing is the process of verifying and validating a software application.
It ensures that the application works according to business requirements.
Testing helps identify defects, bugs, and issues before the software is released.
It improves software quality, reliability, and performance.
Testing reduces production failures and maintenance costs.
It increases customer satisfaction by delivering a stable product.


2. Differentiate between Manual Testing and Automation Testing.
Answer:
Manual Testing is performed by a human tester without using automation tools.
Automation Testing uses tools like Playwright to execute test cases automatically.
Manual testing is suitable for exploratory and usability testing.
Automation testing is ideal for repetitive and regression testing.
Automation provides faster execution and higher accuracy.
Automation saves time and effort for large-scale projects.


3. Explain Smoke Testing and Sanity Testing.

Answer:

Smoke Testing verifies whether the major functionalities of an application are working.
It is performed on every new build before detailed testing begins.
Sanity Testing checks specific functionality after a bug fix or minor change.
Smoke Testing covers the overall application flow.
Sanity Testing focuses only on affected modules.
Both help determine whether further testing should continue.


4. What is Regression Testing?

Answer:

Regression Testing ensures that existing functionality works after code changes.
It verifies that new changes have not introduced new defects.
It is commonly automated using Playwright.
Regression suites are executed before releases.
It improves application stability and reliability.
It reduces deployment risks.


5. What is End-to-End Testing?

Answer:

End-to-End Testing validates complete business workflows.
It simulates real user actions across the application.
It verifies integration between frontend and backend systems.
It tests database interactions and API communication.
It ensures the application behaves correctly from start to finish.
It provides confidence before production deployment.


6. What is Playwright?

Answer:

Playwright is an open-source automation framework developed by Microsoft.
It supports Chromium, Firefox, and WebKit browsers.
It is used for UI, API, and End-to-End testing.
It provides built-in auto-waiting functionality.
Playwright supports parallel execution.
It works effectively with modern web applications like React and Angular.


7. What are the advantages of Playwright over Selenium?

Answer:

Playwright provides built-in auto-waiting.
It supports multiple browsers using a single API.
It executes tests faster than Selenium.
It includes powerful debugging tools like Trace Viewer.
It handles modern JavaScript applications efficiently.
It reduces flaky tests and improves stability.


8. Explain Browser Architecture.

Answer:

Browser Architecture defines how browsers process and display web pages.
The Browser Process manages tabs and user interactions.
The Renderer Process renders webpage content.
The Network Process handles HTTP requests and responses.
The GPU Process handles graphics rendering.
Understanding browser architecture helps troubleshoot automation issues.


9. Describe the HTTP Request and Response Lifecycle.

Answer:

The browser sends an HTTP request to the server.
The server receives and processes the request.
Business logic and database operations are executed.
The server generates an HTTP response.
The browser receives and renders the response.
The Network tab can be used to monitor this lifecycle.


10. What is DOM and why is it important for automation?

Answer:

DOM stands for Document Object Model.
It represents a webpage as a tree structure.
Every HTML element becomes a node in the DOM.
Automation tools interact with DOM elements.
Locators are created using DOM information.
Understanding DOM helps build reliable automation scripts.


11. What are Locators in Playwright?

Answer:

Locators are used to identify web elements on a page.
Playwright interacts with elements using locators.
Locators are required for actions such as click and fill.
Good locators improve script stability.
Playwright provides built-in locator strategies.
Reliable locators reduce maintenance effort.


12. Explain ID Locator and Name Locator.

Answer:

ID Locator identifies elements using the id attribute.
Name Locator identifies elements using the name attribute.
IDs are usually unique within a webpage.
Name attributes are commonly used in forms.
Both locator types are simple and efficient.
They improve script readability and performance.


13. What are CSS Selectors?

Answer:

CSS Selectors are patterns used to locate HTML elements.
They are faster than XPath in many cases.
CSS can target elements using ID, class, and attributes.
CSS selectors are easy to read and maintain.
They are widely used in automation frameworks.
Complex UI structures can also be handled using CSS.


14. What is XPath?

Answer:

XPath is a language used to locate elements in XML and HTML documents.
It navigates through the DOM structure.
XPath can identify elements without IDs or classes.
It supports parent-child and sibling relationships.
XPath is useful for complex element identification.
It is commonly used in automation testing.


15. What is Relative XPath?

Answer:

Relative XPath starts searching from any node in the DOM.
It is more flexible than Absolute XPath.
It does not depend on the complete DOM path.
Relative XPath is less likely to break after UI changes.
It improves locator maintainability.
It is widely recommended in automation projects.


16. Why are Playwright Recommended Locators preferred?

Answer:

They are based on user-facing attributes.
They improve script readability.
They provide better stability than XPath.
They are less affected by UI structure changes.
They align with accessibility standards.
Playwright officially recommends them as best practice.

18. Explain getByText().

It mimics how users identify elements.

19. Explain getByLabel().
